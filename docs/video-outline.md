# 视频素材提纲

## 标题候选

- 不买罗技键盘，也让 Logi Flow 的键盘跟着鼠标走
- 我给 Logi Flow 补了一个键盘桥
- 普通键盘如何跨电脑跟随 Logi Flow

## 视频主线

### 1. 问题开场

画面：

- 展示 Logi Flow 鼠标可以跨电脑移动。
- 然后展示普通键盘不会跟随。

旁白：

```text
Logi Flow 很好用，但它的键盘跟随基本绑定罗技生态。
我不想为了这个再买一把键盘，于是做了一个旁路工具：
Flow 继续管鼠标，我自己写一个键盘桥。
```

### 2. 最小目标

画面：

- 简单图示：Host 主电脑、Remote 副电脑、TCP 连接。

旁白：

```text
第一版目标非常小：主电脑捕获键盘，副电脑接收并模拟输入。
不破解 Logi Flow，不接入它的私有协议，只做一个独立工具。
```

### 3. 第一次跑通

画面：

- 主电脑窗口显示 Host。
- 副电脑窗口显示 Remote。
- 副电脑记事本出现 `hello123`。

旁白：

```text
能把字打到副电脑上，说明链路跑通了。
但真正做成自己能每天用的工具，问题才刚开始。
```

### 4. 连接坑

画面：

- 展示找不到服务、扫错网段、连接超时的日志。

旁白：

```text
第一个坑是网络。
IP 会填错，防火墙会挡，UDP 广播也不一定通。
甚至代理虚拟网卡会让程序扫到 198.18 这种错误网段。
```

字幕关键词：

- 防火墙
- 自动发现
- 虚拟网卡
- 局域网扫描

### 5. Windows 输入坑

画面：

- 展示 `WinError 87 参数错误`。
- 展示 `INPUT size 32 -> 40` 的对比。

旁白：

```text
第二个坑是 Windows 的 SendInput。
64 位 Windows 的 INPUT 结构体要 40 字节。
我一开始写成 32 字节，于是系统只返回一个很抽象的参数错误。
```

### 6. 从能用到好用

画面：

- 命令行版切到 GUI 版。
- 双击 Host / Remote。

旁白：

```text
命令行能测试，但不是给自己长期用的。
所以我把它做成双击版：主电脑开 Host，副电脑开 Remote，不用填 IP。
```

### 7. 输入隔离

画面：

- remote 模式下副电脑输入，本机不再输入。

旁白：

```text
只监听键盘还不够，因为主电脑也会收到按键。
最后 Host 改成低级键盘钩子：remote 模式下吞掉本机按键，只转发给副电脑。
```

### 8. 自动跟随鼠标

画面：

- 鼠标移动到副电脑，Host 显示 `target: remote`。
- 鼠标回主电脑，Host 显示 `target: local`。

旁白：

```text
最后做了一个近似的自动跟随：
副电脑鼠标动了，就认为人在副电脑。
主电脑鼠标动了，就切回主电脑。
这不是读取 Logi Flow 状态，但足够贴近日常使用。
```

### 9. 结尾

旁白：

```text
这个工具不是要替代 Logi Flow，而是给它补一块缺失的普通键盘跟随能力。
最有意思的是，这种小工具不是一开始就完美设计出来的，
而是在一次次真实手测、报错、修复里慢慢变得可用。
```

## 可拍素材清单

- Logi Flow 鼠标跨屏演示。
- 普通键盘不能跟随的原始问题。
- Host / Remote GUI 双击启动。
- Remote 自动搜索日志。
- 防火墙放行规则。
- `WinError 87` 错误截图。
- 副电脑记事本输入成功。
- remote 模式下主电脑不再被输入。
- 鼠标回主电脑后键盘回本机。

## 适合剪成短视频的三段

1. 问题：Logi Flow 鼠标能走，键盘不能走。
2. 过程：IP、防火墙、SendInput、输入隔离几个坑。
3. 结果：双击两个 exe，普通键盘也能跟随。
