# 视频脚本提纲：给 Logitech Flow 补一个普通键盘跟随

## 标题候选

- 不买罗技键盘，也让普通键盘跟随 Logitech Flow
- 我给 Logitech Flow 补了一个键盘桥
- 普通键盘如何跨电脑跟随鼠标

## 一句话简介

Logitech Flow 可以让鼠标跨电脑移动，但普通键盘不会跟随。Devices Router 的做法是：Flow 继续管鼠标，我们自己写一个键盘桥，把主电脑键盘输入送到副电脑。

## 结构

### 1. 开场：痛点

画面：

- 鼠标从主电脑移动到副电脑。
- 尝试用普通键盘输入，发现字还在主电脑。

旁白：

```text
Logitech Flow 很好用，但键盘跟随基本绑定罗技生态。我不想为了这个再买一把键盘，所以做了一个旁路工具：鼠标还是 Flow 管，键盘我自己转发。
```

### 2. 最小目标

画面：

- 简单示意图：Host 主电脑、Remote 副电脑、TCP 连接。

旁白：

```text
目标不是破解 Flow，也不是重写完整 KVM。第一版只做一件事：主电脑抓键盘，副电脑接收并模拟输入。
```

### 3. 第一次跑通

画面：

- 主电脑显示主电脑模式。
- 副电脑显示副电脑模式并连接成功。
- 副电脑记事本出现 `hello123`。

旁白：

```text
当字第一次出现在副电脑记事本里，说明核心链路跑通了。但真正做成每天可用的软件，问题才刚开始。
```

### 4. 网络坑

画面：

- 连接失败、找不到主电脑、扫描网段日志。
- 防火墙或端口说明。

旁白：

```text
最先遇到的是网络问题。IP 会填错，防火墙会挡，UDP 广播也不一定通。甚至代理软件的虚拟网卡会让程序扫到错误网段。
```

关键词：

- 自动发现
- 局域网扫描
- 防火墙
- 虚拟网卡

### 5. Windows 输入坑

画面：

- 展示 `WinError 87 参数错误`。
- 展示 `SendInput` / `INPUT` 结构。

旁白：

```text
第二个坑是 Windows SendInput。64 位 Windows 的 INPUT 结构体大小必须对，写错之后系统只给一个很抽象的参数错误。
```

### 6. 从命令行到桌面软件

画面：

- 旧命令行版本切到 Tauri 桌面版。
- 展示浅色控制台、日志、状态、更新页。

旁白：

```text
命令行能验证想法，但不是普通人愿意长期使用的工具。所以后面迁到了 Tauri/Rust，做成一个更像正式桌面软件的控制台。
```

### 7. 输入隔离

画面：

- 键盘目标切到副电脑时，主电脑不再输入。
- 副电脑输入框收到文字。

旁白：

```text
只监听键盘还不够。切到副电脑时，主电脑本地也会收到输入。最后用低级键盘 hook 实现：remote 模式下吞掉主电脑输入，只转发给副电脑。
```

### 8. 鼠标在哪，键盘去哪

画面：

- 鼠标在副电脑动，键盘目标变成副电脑。
- 鼠标回主电脑动，键盘目标变回主电脑。

旁白：

```text
这个工具没有读取 Flow 私有协议，而是用一个近似判断：副电脑鼠标动了，就认为人在副电脑；主电脑鼠标动了，就切回主电脑。
```

### 9. 自动更新和排障

画面：

- 副电脑自动更新日志。
- 清空日志、复制日志、连接状态心跳。

旁白：

```text
为了让它给别人用，后面加了局域网自动更新、心跳检测和日志按钮。这样出了问题，不用再截图一堆命令行，直接复制日志就能排查。
```

### 10. 结尾

旁白：

```text
这个项目最有意思的地方，不是一次性设计出完美架构，而是在真实使用里不断遇到小问题：连不上、输入不了、更新失败、状态不准，然后一层层把它补成一个能日常使用的小工具。
```

## 短视频三段式

1. 问题：鼠标能跨电脑，普通键盘不能。
2. 过程：写一个键盘桥，踩了网络、输入注入、隔离、更新几个坑。
3. 结果：两台电脑打开软件，鼠标在哪边，键盘就跟到哪边。
