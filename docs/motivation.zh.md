# 项目动机：一点很真实的碎碎念

这个工具最开始不是为了做一个宏大的 KVM，也不是为了挑战 Logitech Flow。

更真实的需求是：

```text
我想在 vibe coding 的时候，把另一台电脑当开发机用；
同时这台主电脑还可以打 LOL、看东西、处理自己的窗口。
鼠标可以靠 Logitech Flow 过去，但键盘跟不过去。
那就自己补一个键盘跟随。
```

所以 Devices Router 的定位一直很克制：

- 不替代 Logitech Flow。
- 不破解 Flow 私有协议。
- 不伪装 Logitech 设备。
- 不做完整远程桌面。
- 只补一块最烦人的缺口：普通键盘跟着鼠标去另一台电脑。

这个出发点也解释了为什么它会优先追求这些体验：

- 双击就能用，不想每次敲命令。
- 副电脑自动找主电脑，不想记 IP。
- 鼠标去哪边，键盘就去哪边。
- 副电脑自动更新，不想每次手动复制 exe。
- 日志能复制、能清空，方便边测边修。

它本质上是一个“让两台电脑在日常使用里更顺手”的小工具。开发机继续开发，娱乐机继续娱乐，中间不用为了几行输入在两把键盘之间来回切。

这也是这个项目比较有意思的地方：它不是从一个漂亮架构开始的，而是从一个很生活化的懒人需求开始，一边真实使用，一边把连接、输入、更新、状态显示这些细节慢慢补齐。
