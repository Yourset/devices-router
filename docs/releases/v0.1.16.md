# Devices Router v0.1.16

## Changes

- Added an administrator-permission status line in the desktop overview.
- Added an in-app "restart as administrator" action for controlling elevated PowerShell, Terminal, IDE, or other admin windows on the target PC.
- Added a warning prompt when Devices Router is not running as administrator, so permission-related input failures are easier to identify.

## Notes

Windows blocks normal apps from injecting input into elevated windows. If the target window is running as administrator, run Devices Router as administrator on that same PC too.
