# Devices Router v0.1.19

## 中文

### 修复

- 修复切到副电脑后，主电脑偶发残留 `Ctrl`、`Alt`、`Shift`、`Win` 状态的问题。
- 修复残留修饰键导致回到主电脑后普通按键变成系统快捷键的问题，例如按 `A` 触发截图、搜索或其它快捷键。
- 修复 uTools 等快捷面板里上下方向键不生效的问题。

### 原因

- 切换到副电脑时，主电脑会开始拦截键盘输入。如果切换快捷键的 key-down 已经被 Windows 收到，但 key-up 被 DR 拦截，Windows 可能会误以为 `Ctrl`、`Alt` 等修饰键仍然按着。
- uTools 这类面板对方向键事件更挑，不一定接受普通 virtual-key 注入。方向键需要更接近真实键盘的 `scan code + extended key` 事件。

### 改动

- 切到副电脑前，DR 会主动给主电脑补发本地修饰键松开事件。
- 上下左右、Home/End、PageUp/PageDown、Insert/Delete 等导航键改为 scan-code based extended-key 注入。
- 保留普通文字和输入法相关按键的 virtual-key 路径，避免回到早期 Unicode 主路径导致输入法异常。

### 排查建议

- 两端都需要更新到 `v0.1.19` 或更高。
- 如果 uTools、PowerShell、Terminal、IDE 等目标程序是管理员身份运行，副电脑上的 DR 也要用管理员身份运行。
- 副电脑日志里出现 `已输入按键：<38>` 或 `已输入按键：<40>`，说明 DR 已经完成上下键注入；如果目标程序仍不响应，要继续查焦点或权限。

## English

### Fixes

- Fixed occasional stuck `Ctrl`, `Alt`, `Shift`, or `Win` state on the host after switching control to the secondary PC.
- Fixed normal keys turning into system shortcuts after returning to the host, such as `A` triggering screenshot, search, or another shortcut.
- Fixed arrow keys in uTools-style launcher panels.

### Cause

- When switching control to the secondary PC, Devices Router starts suppressing host keyboard input. If Windows receives the switch shortcut key-down but the key-up is suppressed, Windows can keep a modifier logically pressed.
- uTools-style panels are stricter than normal text fields. They may ignore plain virtual-key arrow events and require physical-key-like `scan code + extended key` metadata.

### Changes

- Devices Router now sends local modifier key-up events before suppressing host input for remote control.
- Arrow keys, Home/End, PageUp/PageDown, Insert/Delete, and related navigation keys now use scan-code based extended-key injection.
- Normal text and IME-sensitive keys stay on the virtual-key path, avoiding the earlier Unicode-primary behavior that could break IME state.

### Diagnostics

- Both PCs should run `v0.1.19` or newer.
- If uTools, PowerShell, Terminal, an IDE, or another target app runs as administrator, run Devices Router as administrator on the same remote PC.
- If the remote log shows `input key: <38>` or `input key: <40>`, Devices Router has injected the up/down key. If the target app still does not respond, check focus or elevation next.
