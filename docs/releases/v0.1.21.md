# Devices Router v0.1.21

## 中文

### 修复

- 修复副电脑 UI 中“键盘目标”偶尔不同步的问题。
- 副电脑发出键盘切换请求后，现在会等待主电脑确认最终目标。
- 主电脑每次处理副电脑请求或鼠标活动切换后，会回传当前键盘目标状态。

### 原因

- 旧协议只有 `TargetRequest`，也就是副电脑只会“请求主电脑切换”。
- 主电脑切换成功后没有把最终状态同步回副电脑。
- 因此两边 UI 可能各自显示本地状态：主电脑实际已经切到副电脑，但副电脑界面还停留在旧值，或者反过来。

### 改动

- 新增协议消息 `TargetState`。
- 主电脑在目标变化后发送 `TargetState`。
- 副电脑收到 `TargetState` 后用主电脑确认值更新本地 UI。
- 副电脑发请求时会记录“正在等待主电脑确认键盘目标”，方便看日志判断链路。

## English

### Fixes

- Fixed occasional keyboard-target UI mismatch on the secondary PC.
- After the secondary PC sends a target switch request, it now waits for host confirmation.
- The host now sends its current keyboard target after handling a remote request or mouse-activity switch.

### Cause

- The old protocol only had `TargetRequest`, meaning the secondary PC could request a switch but did not receive the final host state.
- After the host switched target, it did not confirm that state back to the secondary PC.
- As a result, both UIs could show their own local state instead of the same confirmed target.

### Changes

- Added `TargetState` protocol message.
- Host sends `TargetState` after target changes.
- Remote updates local UI from the host-confirmed target.
- Remote logs that it is waiting for host confirmation after sending a switch request.
