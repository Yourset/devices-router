# Devices Router v0.1.22

## 中文

### 修复

- 修复回车在部分副电脑窗口里不响应的问题。
- 扩大功能键注入范围：回车、Tab、Esc、Backspace、空格、CapsLock、PrintScreen、F1-F12 等非文本键统一使用 scan-code 注入。
- 保留字母、数字和常见标点的 virtual-key 路径，避免破坏普通输入和输入法行为。

### 原因

- 之前只有方向键、Insert/Delete、Home/End、PageUp/PageDown 和修饰键会走 scan-code。
- 回车、Tab、Esc、Backspace 等仍走普通 virtual-key 注入。
- 一些程序会更严格地区分真实键盘输入和普通虚拟键事件，所以这类功能键可能在特定窗口里不生效。

### 验证

- 新增 `functional_keys_use_scan_codes` 回归测试。
- 确认普通字母和数字仍保留原 virtual-key 路径。

## English

### Fixes

- Fixed Enter not working in some secondary-PC windows.
- Expanded scan-code injection to functional keys: Enter, Tab, Esc, Backspace, Space, CapsLock, PrintScreen, F1-F12, and related non-text keys.
- Kept letters, digits, and common punctuation on the virtual-key path to avoid disrupting normal typing and IME behavior.

### Cause

- Previous versions only used scan-code injection for navigation keys, Insert/Delete, Home/End, PageUp/PageDown, and modifiers.
- Enter, Tab, Esc, Backspace, and similar keys still used plain virtual-key injection.
- Some applications distinguish physical-like keyboard events from plain virtual-key events, so those keys could fail in specific windows.

### Verification

- Added the `functional_keys_use_scan_codes` regression test.
- Confirmed regular letters and digits still stay on the virtual-key path.
