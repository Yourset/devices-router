# Devices Router v0.1.20

## 中文

### 修复

- 修复副电脑上 `Shift` 不稳定或无法参与大小写切换的问题。
- 改善界面稳定性：降低状态刷新频率，避免用户正在输入主电脑 IP 时界面被频繁重绘打断。
- 改善小窗口布局：窄窗口下改成单列布局，日志区高度随窗口自适应，输入框不再溢出。

### 原因

- `v0.1.19` 已经让方向键走 scan code 注入，但 `Shift/Ctrl/Alt/Win` 等修饰键仍主要走 virtual-key 注入。部分目标程序对修饰键状态更敏感，可能导致 `Shift + 字母` 没有稳定变成大写。
- 前端每 200ms 全量刷新一次界面，虽然能及时显示状态，但会打断输入框编辑，也可能让按钮点击和日志区域显得不稳定。

### 改动

- `Shift/Ctrl/Alt/Win` 等修饰键改为 scan-code 注入，但不误加 extended-key 标志。
- 普通字母、数字仍保留 virtual-key 路径，避免影响 IME。
- 状态刷新从 200ms 调整为 1000ms。
- 用户正在编辑输入框时暂停整页重绘。
- 新增鼠标能力后续规划文档：`docs/mouse-roadmap.zh.md`。

## English

### Fixes

- Fixed unstable `Shift` behavior on the secondary PC, especially when using Shift to type uppercase letters.
- Improved UI stability by reducing status refresh frequency and avoiding full-page re-render while editing the host IP field.
- Improved narrow-window layout with a single-column workspace, adaptive log height, and safer input sizing.

### Cause

- `v0.1.19` moved navigation keys to scan-code injection, but modifiers such as `Shift/Ctrl/Alt/Win` still mostly used virtual-key injection. Some target apps are stricter about modifier state and may not reliably treat `Shift + letter` as uppercase input.
- The frontend refreshed the whole UI every 200 ms. That kept status fresh, but could interrupt text editing or make controls feel unstable.

### Changes

- Modifier keys now use scan-code injection without incorrectly adding the extended-key flag.
- Normal letters and numbers stay on the virtual-key path to avoid disturbing IME behavior.
- Status refresh interval changed from 200 ms to 1000 ms.
- Full-page re-render is skipped while an input field is focused.
- Added the mouse capability roadmap: `docs/mouse-roadmap.zh.md`.
