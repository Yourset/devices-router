# Devices Router v0.2.4

## 中文

- 所有控制 TCP 连接启用 `TCP_NODELAY`，减少局域网小包等待。
- 鼠标活动新增 UDP `8766` 快速通道；鼠标本身仍由 Logi 处理，Devices Router 不移动、不拦截、不转发鼠标。
- 副电脑按 `0/5/15ms` 三次发送同一活动包，主电脑使用设备 ID、会话令牌、来源 IP、连接代次和活动编号校验并去重。
- UDP 未就绪或切换未在 60ms 内确认时自动通过 TCP 补发；目标代次不一致的迟到补发会被拒绝。
- RTT 改为主电脑统一探测，副电脑显示主电脑同步的结果，避免双向探测重复写入。
- 主界面显示活动通道、实时 RTT、稳定 RTT、抖动和近 20 次丢包；这些值局部更新，不重建当前面板。
- 保留 `latencyMs` / `hostLatencyMs` 兼容字段并映射到稳定 RTT。
- `v0.2.3` 副电脑可继续通过 TCP 连接，并使用现有局域网自动更新升级。

## English

- Enables `TCP_NODELAY` on every control TCP connection to reduce small-packet delay.
- Adds a UDP `8766` mouse-activity fast path. Logi still owns the pointer; Devices Router never moves, captures, or forwards mouse input.
- A remote sends each activity at `0/5/15ms`; the host validates and deduplicates by device ID, session token, source IP, connection generation, and activity ID.
- Before UDP is ready, or when a target change is not confirmed within 60ms, activity falls back to TCP. Stale target-epoch fallbacks are rejected.
- RTT probes are host-authoritative and synchronized to the remote, eliminating duplicate bidirectional probes.
- The overview shows activity transport, live RTT, stable RTT, jitter, and loss over the latest 20 attempts using local field updates.
- Legacy `latencyMs` and `hostLatencyMs` remain mapped to stable RTT for compatibility.
- v0.2.3 remotes remain compatible over TCP and can auto-update from the host.
