# Devices Router 使用教程

## 适合谁

你想用任意普通键盘和鼠标控制两台 Windows 电脑，不想依赖 Logitech Flow 或罗技硬件。Devices Router 会把主电脑的键盘鼠标直接转发到副电脑。

推荐场景：

- 主电脑插着实体键盘。
- 副电脑需要偶尔输入文字、快捷键或代码。
- 希望鼠标划过屏幕边缘时，键盘也跟着切换。
- 不想为了跨屏控制购买指定品牌硬件。

## 安装和启动

普通用户只需要安装包，不需要 Node.js、Rust、Python，也不需要命令行。

1. 从 Release 下载 `Devices Router_版本号_x64-setup.exe`。
2. 主电脑安装一次。
3. 副电脑安装一次。
4. 两台电脑打开同一个 `Devices Router`。

主电脑：

1. 打开软件。
2. 点击 `主电脑模式`。
3. 保持软件运行。

副电脑：

1. 打开软件。
2. 点击 `副电脑模式`。
3. 等待状态变成 `已连接`。
4. 打开目标输入框，例如记事本、聊天框、浏览器输入框或 IDE。

## 日常使用

正常情况下不需要手动点按钮：

- 从主电脑右边缘继续向右，键盘鼠标一起进入副电脑。
- 把副电脑光标推到左边缘后继续向左，键盘鼠标一起回主电脑。

如果自动判断不符合预期，可以手动切换：

- 点击界面里的 `键盘到主电脑`
- 点击界面里的 `键盘到副电脑`
- 在主电脑按 `Ctrl+Alt+1` 回主电脑
- 在主电脑按 `Ctrl+Alt+2` 到副电脑

## 安全释放

如果鼠标没有正常划回来，直接在主电脑按 `Ctrl+Alt+Esc`。这个组合键由主电脑本地处理，不会转发给副电脑，也不依赖网络是否正常。

还可以使用：

- 软件首页或“鼠标跟随”页的 `立即回到主电脑`
- Windows 托盘菜单的 `立即释放控制（Ctrl+Alt+Esc）`

连接中断、远端退出或鼠标消息发送失败时，主电脑也会自动解除本地键盘和鼠标拦截。

## 看懂状态

首页里最重要的是三项：

- `模式`：当前软件在主电脑模式还是副电脑模式。
- `连接`：两台电脑之间是否已经建立连接。
- `键盘目标`：主电脑键盘当前应该发给哪台电脑。

如果你在副电脑动鼠标，但 `键盘目标` 没变成副电脑，说明鼠标活动上报或控制消息没有通。

## 日志按钮

日志区有三个按钮：

- `清空日志`：清掉当前窗口里的历史日志。
- `复制日志`：把日志复制到剪贴板，方便发给开发者排查。
- `导出日志`：保存成 `.txt` 文件。

排查问题时，优先看这些关键词：

- `已连接主电脑`
- `切换请求已发出`
- `副电脑请求：键盘到副电脑`
- `已转发按键`
- `已输入按键`
- `心跳检测失败`

## 自动更新

主电脑会提供局域网更新服务。副电脑连接成功后，会自动检查主电脑上的更新包。

通常你只需要更新主电脑；副电脑会在连上后自动跟随更新。更新失败时，日志会显示下载、校验或安装失败的原因。

## 开机自启动

在 `更新` 页里可以打开 `开机自动启动`。软件会记住上次模式：

- 上次是主电脑模式，下次启动后继续主电脑模式。
- 上次是副电脑模式，下次启动后继续副电脑模式。

## 最短测试流程

1. 主电脑打开 `主电脑模式`。
2. 副电脑打开 `副电脑模式`。
3. 副电脑状态显示 `已连接`。
4. 副电脑打开记事本。
5. 从主电脑右边缘向右划入副电脑。
6. 用主电脑键盘输入 `hello123`。
7. 把副电脑光标推到左边缘并继续向左，确认鼠标回到主电脑。
8. 再次划入副电脑，按 `Ctrl+Alt+Esc`，确认主电脑在断网前提下也能立即恢复控制。

## 使用边界

这个工具不是硬件 KVM，也不是 Logitech Flow 插件。它通过局域网和 Windows 输入接口实现独立的键盘鼠标转发。

UAC 弹窗、管理员窗口、部分游戏或被安全软件保护的窗口可能不接受模拟输入，这是 Windows 安全边界导致的。
