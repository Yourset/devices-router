# H5 副电脑客户端说明

## 为什么不是纯 H5

浏览器页面不能把按键注入到 Windows 的其他程序里。这是浏览器安全模型限制，不是实现难度问题。

所以当前设计是：

- H5 页面：负责显示状态、日志、停止按钮和后续设置入口。
- 本地 helper：负责自动发现主电脑、接收键盘事件、监听副电脑鼠标移动、调用 Windows 输入 API。

这样用户看到的是更容易理解的网页界面，同时核心输入能力仍然可用。

## 启动流程

1. 用户双击 `FlowKeyboardRemote.exe`。
2. 程序启动本地 HTTP 服务，监听 `127.0.0.1` 的随机端口。
3. 程序打开默认浏览器进入 H5 控制台。
4. 后台线程运行原有副电脑客户端逻辑。
5. H5 页面轮询 `/api/logs` 获取连接、更新、错误日志。

## 当前接口

- `GET /`：H5 控制台页面
- `GET /api/logs?after=<id>`：获取增量日志
- `GET /api/stop`：停止客户端
