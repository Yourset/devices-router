# Flow Keyboard Bridge 技术复盘

## 架构

```text
Host 主电脑
  - 低级键盘钩子捕获按键
  - remote 模式下吞掉本机按键
  - 通过 TCP 8765 发送按键事件
  - UDP 8766 广播自身
  - 轮询本机鼠标位置，鼠标回来时切 local

Remote 副电脑
  - 自动发现 Host
  - TCP 连接 Host
  - 接收按键事件
  - 使用 SendInput 注入按键
  - 监听副电脑鼠标移动，通知 Host 切 remote
```

## 关键端口

- TCP `8765`：键盘事件和控制事件。
- UDP `8766`：Host 自动发现广播。

## 关键问题和解决

| 问题 | 表现 | 根因 | 修复 |
| --- | --- | --- | --- |
| IP 填错 | Remote 连接后异常或连不上 | 主电脑实际是 `192.168.31.18`，不是 `192.168.1.23` | 增加自动发现和扫描 |
| 防火墙拦截 | Remote 超时 | TCP/UDP 入站未放行 | 放行 TCP 8765 / UDP 8766 |
| 自动发现扫错网段 | 扫描 `198.18.0.0/24` | Mihomo/TUN 虚拟网卡被当成 LAN | 忽略 `198.18.0.0/15` |
| Remote 5 秒断线 | 已连接后反复重连 | socket 连接超时没有清掉 | 连接成功后 `settimeout(None)` |
| `WinError 87` | 副电脑按键注入失败 | 64 位 Windows `INPUT` 结构体大小错误 | 补齐 union，确认 `sizeof(INPUT) == 40` |
| 主电脑也收到输入 | remote 模式下双边输入 | 普通监听不能拦截本机按键 | 改用低级键盘钩子 |
| 鼠标回主机但键盘不回 | 键盘仍在副电脑 | Logi Flow 回来不一定触发鼠标事件 | Host 用 `GetCursorPos` 轮询 |

## 为什么不接入 Logi Flow

Logi Flow 不是开放协议。直接适配它可能涉及：

- 私有协议逆向。
- 设备身份模拟。
- 与 Logi Options+ 的兼容风险。
- 未来版本更新后失效。

当前方案选择旁路：

- Logi Flow 继续管鼠标。
- Flow Keyboard Bridge 只管键盘。
- 通过鼠标活动推断当前目标电脑。

## 当前限制

- 主要面向 Windows -> Windows。
- 管理员窗口、UAC、部分游戏可能不接受普通模拟输入。
- 中文输入法组合状态可能需要继续优化。
- 鼠标自动跟随是启发式判断，不是真正读取 Logi Flow 状态。
- 若 Host 程序崩溃，低级键盘钩子会随进程退出释放，但仍建议保留 `Ctrl+Alt+1` 作为安全回本机路径。
