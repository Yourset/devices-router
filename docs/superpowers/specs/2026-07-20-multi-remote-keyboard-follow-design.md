# Devices Router v0.2.0 多副电脑键盘跟随设计

## 目标

让一台主电脑同时连接两台副电脑。Logi 继续负责鼠标跨电脑；Devices Router 只观察三台电脑上的鼠标活动，并把主电脑键盘切换到最后活动的电脑。

第一版界面最多展示两台副电脑，但底层使用设备列表和设备 ID，不把核心结构写死为 `remote1`、`remote2`，为以后扩展保留空间。

## 用户行为

- 主电脑鼠标活动时，键盘立即回主电脑。
- 副电脑 A 鼠标活动时，键盘切到副电脑 A。
- 副电脑 B 鼠标活动时，键盘切到副电脑 B。
- 多台电脑近乎同时活动时，最后活动者获得键盘，并沿用 `30ms` 切换防抖。
- 鼠标不由 Devices Router 转发、拦截或移动，继续交给 Logi。
- `Ctrl+Alt+Esc` 在任何状态下都强制回主电脑并释放所有远端按键。
- 界面显示主电脑、两台副电脑的名称、在线状态和当前键盘目标。

## 设备身份与配置

- 每台副电脑首次运行时生成永久 `deviceId`，保存到本机配置；IP 变化或程序重启后仍视为同一设备。
- 默认设备名称读取 Windows 电脑名，并允许在 Router 中设置仅用于显示的别名。
- 旧配置中的单个 `remoteHost` 保留兼容读取，用作第一台副电脑或发现失败时的连接提示，不再作为主电脑唯一的远端状态来源。
- 主电脑最多接受两台不同设备在线。第三台连接返回明确拒绝原因，不替换现有设备。

## 协议与连接

- `ClientHello` 增加可选的 `deviceId`、`deviceName` 和协议能力字段。可选字段保证新主电脑仍能接受旧副电脑，旧主电脑也能忽略新字段。
- 主电脑的 TCP listener 持续接受连接，每台副电脑由独立会话处理；单台断线不能阻塞或结束其他会话。
- 主电脑维护线程安全的会话注册表：设备身份、连接代次、发送通道、最近活动时间和在线状态。
- 同一 `deviceId` 重连时，新会话替换旧会话；旧会话稍后结束时不得误删新会话。
- 每台会话保留现有心跳。当前目标断线、超时或发送失败时，立即释放远端输入并回到主电脑。
- 升级顺序为主电脑优先，然后两台副电脑通过现有局域网更新升级。兼容期内，缺少设备 ID 的旧副电脑只能占用一个 legacy 槽位。

## 键盘与鼠标活动路由

- 键盘目标从二元 `Local/Remote` 升级为 `Local/Device(deviceId)`；对前端序列化为 `local` 或具体设备 ID。
- 主电脑只运行一套全局键盘监听。键盘事件只发送给当前目标设备的会话，不广播给其他副电脑。
- 每台副电脑沿用本地鼠标活动采样，并发送带设备身份的活动事件。主电脑本地鼠标活动直接参与同一套“最后活动者”仲裁。
- 从一个远端切到另一个远端前，必须向旧目标补发全部按键和鼠标按钮松开，再更新目标，避免 `Ctrl/Shift/Alt` 粘键。
- 切换、断线、紧急释放都通过同一个安全切换入口执行，不能各自维护不一致的释放逻辑。

## 状态与界面

- `AppStatus` 增加设备列表、每台设备在线状态和 `activeDeviceId`；总览页不再使用单一 `connected` 表达全部连接。
- 自动跟随页显示三台电脑当前状态、最后活动时间和当前目标，并保留手动切到主电脑/副电脑的按钮作为诊断与脱困手段。
- 网络诊断按设备显示地址、键盘端口和更新端口状态；两台设备都不能让主界面产生页面滚动，继续遵守现有滚动白名单。
- 托盘状态至少区分：未连接、已连接但键盘在本机、键盘在某台副电脑；托盘提示显示当前目标名称。

## 故障处理

- 第三台不同设备连接：拒绝并记录“已达到两台副电脑上限”。
- 重复设备连接：新连接接管，旧连接失效，不占两个名额。
- 当前目标断线或休眠：立即回主电脑、取消键盘抑制、释放修饰键。
- 非当前目标断线：仅更新该设备状态，不改变当前键盘目标。
- 两台鼠标持续同时活动：按最新活动时间决定目标，并受 `30ms` 防抖约束，避免快速往返。
- 无法识别设备的旧客户端：允许占用一个 legacy 槽位，但不允许同时接入两个无法区分的旧客户端。

## 测试与验收

- 单元测试：设备注册、两设备并存、第三台拒绝、重复设备代次保护、目标断线回本机、非目标断线不切换。
- 路由测试：主电脑、设备 A、设备 B 依次活动；近同时活动最后者胜；防抖期间不来回跳。
- 输入安全测试：远端 A 按住修饰键切到 B、B 断线、紧急释放，均不会留下粘键或持续抑制。
- 协议测试：新旧 `ClientHello` 互通、缺失可选字段、未知能力字段和 legacy 槽位限制。
- 界面契约测试：两台设备状态完整可见，`880×680` 下无页面滚动和控件裁切。
- 实机验收：主电脑加两台副电脑，在同一局域网中连续切换至少 10 分钟；三台鼠标活动均能在可感知的短延迟内正确接管键盘。

## 发布与交付

- 新版本号为 `v0.2.0`，表示连接模型和协议能力的升级。
- 先在主电脑安装 `v0.2.0`，再把同一安装包复制到一台新副电脑安装；原副电脑可通过局域网更新或同一安装包升级。
- 交付时提供 NSIS 安装包绝对路径、SHA256、主电脑安装结果和新电脑最短安装步骤。
