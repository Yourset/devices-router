# 排障手册

## 先确认版本

两台电脑最好都运行同一个版本。当前主线版本是 `v0.2.3`。

在软件首页看版本号。如果副电脑版本落后，先让它连上主电脑并等待自动更新。局域网更新依赖主电脑提供更新包，所以主电脑也必须已经更新到新版本。

## 副电脑找不到主电脑

表现：

- 副电脑一直显示未连接。
- 日志反复出现自动寻找、广播失败、本地扫描失败。

检查顺序：

1. 主电脑是否已经点了 `主电脑模式`。
2. 两台电脑是否在同一个 Wi-Fi 或局域网。
3. Windows 防火墙是否允许 TCP `8765`、TCP `8767`、UDP `8766`。
4. 是否开了 VPN、代理 TUN、虚拟网卡。
5. 副电脑 `网络诊断` 页是否手动填了错误 IP。

可以在副电脑临时手动填写主电脑 IP，例如：

```text
192.168.31.18
```

## 显示已连接，但键盘不过去

按日志链路判断：

1. 副电脑点 `键盘到副电脑`。
2. 副电脑应出现 `切换请求已发出`。
3. 主电脑应出现 `副电脑请求：键盘到副电脑`。
4. 主电脑按键后应出现 `已转发按键`。
5. 副电脑应出现 `已输入按键`。

判断：

- 副电脑没有 `切换请求已发出`：副电脑本地没有可用连接通道。
- 副电脑有发出，但主电脑没请求日志：控制消息没到主电脑。
- 主电脑有请求日志，但没有 `已转发按键`：键盘 hook 或目标状态异常。
- 主电脑有转发，副电脑没有输入：副电脑注入失败，目标窗口没焦点，或目标窗口权限更高。

## 回到主电脑后，普通按键变成快捷键

例如按 `A` 变成截图、搜索或其它快捷键，通常是主电脑残留了 `Ctrl`、`Alt`、`Shift`、`Win` 之类的修饰键状态。

原因是切到副电脑时，DR 会开始拦截主电脑键盘。如果切换快捷键的 key-down 已经被 Windows 收到，但 key-up 被拦截，Windows 可能会以为修饰键还在按住。

`v0.1.20` 起，DR 在切到副电脑前会主动给主电脑补发修饰键松开事件，降低残留概率。如果仍然发生：

1. 先按一下 `Ctrl`、`Alt`、`Shift`、`Win` 并松开，临时恢复 Windows 状态。
2. 确认两端都更新到 `v0.1.22` 或更高。
3. 看主电脑日志里是否出现切换到副电脑的记录。

## uTools 或类似面板里上下键不生效

如果普通文本框、列表可以用上下键，但 uTools 搜索结果里不行，通常不是网络问题，而是目标程序对键盘事件元数据更挑。

早期版本只发送虚拟键码。部分 Electron/快捷面板类程序需要更接近真实键盘的 `scan code + extended key` 事件，才会把方向键当作导航键处理。

`v0.1.19` 起，上下左右、Home/End、PageUp/PageDown、Insert/Delete 等导航键改用 scan-code based extended-key 注入。`v0.1.22` 起，回车、Tab、Esc、Backspace、空格、CapsLock、PrintScreen、F1-F12 等功能键也统一改用 scan-code 注入。如果仍然不生效：

1. 确认副电脑已更新到 `v0.1.22` 或更高。
2. 确认 uTools 窗口有焦点。
3. 如果 uTools 是管理员身份运行，副电脑上的 DR 也要用管理员身份运行。
4. 查看副电脑日志是否出现 `已输入按键：<38>` 或 `已输入按键：<40>`。

## 管理员窗口无法输入

Windows 会阻止普通权限程序向管理员权限窗口注入输入。如果 PowerShell、Terminal、IDE、uTools 等目标窗口是管理员身份运行，副电脑上的 DR 也需要管理员身份运行。

处理方式：

1. 在副电脑 DR 首页查看 `管理员权限` 状态。
2. 如果显示未开启，点击 `以管理员身份重启`。
3. 接受 Windows UAC 提示后，再测试目标窗口输入。

## 移动鼠标后键盘没有自动切换

检查：

- 两端是否都更新到 `v0.2.3`。
- `键盘自动跟随` 页是否显示自动跟随开启。
- 副电脑移动鼠标后，日志是否出现 `已上报鼠标活动给主电脑`。
- 主电脑是否出现 `副电脑鼠标活动：键盘到副电脑`。

这里仅检测鼠标位置是否变化，用来切换键盘；不会转发鼠标坐标、按键或滚轮。自动判断异常时仍可用 `Ctrl+Alt+1/2` 手动切换。

## 连接状态不准

`v0.1.11` 起加入双向心跳。正常情况下，断开后状态会很快变成未连接。

如果界面还显示已连接：

- 点 `清空日志` 后观察新日志。
- 关闭副电脑软件，看主电脑是否出现心跳失败。
- 确认两端都是 `v0.1.11` 或更高版本。

## 更新失败

检查主电脑是否监听更新端口：

```powershell
netstat -ano | Select-String ":8767"
```

检查 manifest：

```powershell
Get-Content "$env:LOCALAPPDATA\Devices Router\updates\manifest.json"
```

常见原因：

- 主电脑没有启动。
- 防火墙挡住 TCP `8767`。
- 更新包文件名、大小或 sha256 与 manifest 不一致。
- 副电脑正在运行旧版，无法自动重启，需要手动重新打开一次。

## 输入法和特殊按键

当前优先保证普通英文键、数字、常用控制键、方向键和基础快捷键。中文输入法组合态、媒体键、游戏键盘宏等仍可能需要继续优化。


## 第二台副电脑无法连接

- 确认三台电脑都已升级到 `v0.2.3`。
- 主电脑最多接受两台不同副电脑；第三台会收到“两台副电脑上限”拒绝原因。
- 旧版副电脑没有设备 ID，只能占用一个 legacy 槽位。先升级主电脑，再逐台升级副电脑。
- 一台副电脑断线不应影响另一台；如果当前目标断线，键盘应立即回主电脑。
