# 排障手册

## 先确认版本

两台电脑最好都运行同一个版本。当前主线版本是 `v0.1.19`。

在软件首页看版本号。如果副电脑版本落后，先让它连上主电脑并等待自动更新。

## 副电脑找不到主电脑

表现：

- 副电脑一直显示未连接。
- 日志里反复出现自动寻找、广播失败、本地扫描失败。

检查顺序：

1. 主电脑是否已经点击 `主电脑模式`。
2. 两台电脑是否在同一个 Wi-Fi/局域网。
3. 主电脑防火墙是否允许 TCP `8765`、TCP `8767`、UDP `8766`。
4. 是否开了 VPN、代理 TUN、虚拟网卡。
5. 副电脑 `网络诊断` 页里是否手动填了错误 IP。

可以在副电脑临时手动填写主电脑 IP，例如：

```text
192.168.31.18
```

## 显示已连接，但键盘不过去

按日志链路判断：

1. 副电脑点击 `键盘到副电脑`。
2. 副电脑应该出现 `切换请求已发出`。
3. 主电脑应该出现 `副电脑请求：键盘到副电脑`。
4. 主电脑按键后应该出现 `已转发按键`。
5. 副电脑应该出现 `已输入按键`。

判断：

- 副电脑没有 `切换请求已发出`：副电脑本地没有可用连接通道。
- 副电脑有发出，但主电脑没有请求日志：控制消息没有到主电脑。
- 主电脑有请求日志，但没有 `已转发按键`：键盘 hook 或目标状态异常。
- 主电脑有转发，副电脑没有输入：副电脑注入失败，或目标输入框没有焦点。

## 鼠标过去后没有自动切换

检查：

- 两端是否都更新到 `v0.1.19` 或更高。
- `鼠标跟随` 页里是否显示自动跟随开启。
- 副电脑移动鼠标后，日志是否出现 `已上报鼠标活动给主电脑`。
- 主电脑是否出现 `副电脑鼠标活动：键盘到副电脑`。

如果副电脑没有上报，说明副电脑鼠标检测线程没有工作。如果副电脑上报了但主电脑没收到，说明连接通道异常。

## 连接状态不准

`v0.1.11` 起加入双向心跳。正常情况下，断开后状态会很快变成未连接。

如果界面还显示已连接：

- 点 `清空日志` 后观察新日志。
- 关闭副电脑软件，看主电脑是否出现心跳失败。
- 确认两端都是 `v0.1.11` 或更高版本。

## 更新失败

检查主电脑是否监听更新端口：

```powershell
netstat -ano | Select-String ":8767"
```

检查 manifest：

```powershell
Get-Content "$env:LOCALAPPDATA\Devices Router\updates\manifest.json"
```

常见原因：

- 主电脑没有启动。
- 防火墙挡住 TCP `8767`。
- 更新包文件名、大小或 sha256 与 manifest 不一致。
- 副电脑正在运行旧版本，无法自动重启，需要手动重新打开一次。

## 输入到了主电脑，不在副电脑

这通常说明键盘目标还是主电脑。先点一次 `键盘到副电脑`，或在主电脑按 `Ctrl+Alt+2`。

如果主电脑仍然收到输入，说明主电脑没有进入 remote 目标，或低级键盘 hook 没正常工作。重启主电脑端软件再试。

## 输入法和特殊按键问题

当前优先保证普通英文键、数字、常用控制键和基础快捷键。中文输入法组合状态、媒体键、游戏键盘宏等可能需要继续优化。
